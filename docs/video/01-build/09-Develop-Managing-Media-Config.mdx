# Managing Media Config overview

This section contains APIs that will help you manage your video and audio config and how to turn them on or off.

  <title>Managing Media Config | Push Video | Push Documentation</title>
</head>

## Enable / Disable Local Media

To enable/disable local media, we'll call the `config()` method on the `aliceVideoCall`.

```typescript
  aliceVideoCall.config({
    video?: Boolean,
    audio?: Boolean
  })
```

---

### Parameters

| Property | Type      | Description                                         |
| -------- | --------- | --------------------------------------------------- |
| `video`  | `Boolean` | `true` to enable video and `false` to disable video |
| `audio`  | `Boolean` | `true` to enable audio and `false` to disable audio |

## How to know the current status of local audio/video?

- The current local video status can be accessed by `data.local.video` similarly for audio -> `data.local.audio`.&#x20;

- If `data.local.video` is true, then that means your local video is enabled, and if it's false then that means the local video is disabled. Likewise for `data.local.audio`.

- Likewise for remote video and audio, you can access the status by `data.incoming[0].video` and `data.incoming[0].audio`.
